Capstone Project ManageX

Project Summary and Vision

ManageX is a comprehensive restaurant management solution aimed at streamlining operations for small and mid-sized restaurant owners. The system integrates a Point-of-Sale (POS) system, inventory tracking, employee scheduling, and customer feedback management into a single, user-friendly platform. The project is developed as part of my capstone course to address real-world challenges faced in the restaurant industry.
Our vision is to empower restaurant owners with an all-in-one digital platform to enhance operational efficiency, reduce human errors, and improve customer satisfaction. ManageX aims to simplify day-to-day tasks, providing actionable insights and automation tools to optimize business performance.

Project / Business Requirements

ManageX is designed to fulfill essential business needs for restaurant owners by integrating multiple modules into a single platform to improve efficiency and accuracy.

  • POS System for billing and order processing
  • Inventory Management with automated alerts
  • Employee Scheduling and Payroll Management
  • Real-time Sales Tracking & Reporting
  • Customer Feedback & Rating Module
  • Role-based Access (Admin, Manager, Employee, Customer)
  • Mobile Application for remote access

Project Plan

The project is structured into multiple phases to ensure systematic development and delivery of a fully functional restaurant management solution.

  • Phase 1: Requirement gathering and competitor analysis
  • Phase 2: System Design (UML diagrams, wireframes, ERD)
  • Phase 3: Backend development (Java + Firebase/SQL)
  • Phase 4: Android App development using Java (Android Studio)
  • Phase 5: Testing (unit, integration, UAT)
  • Phase 6: Deployment and Final Presentation

Requirements Analysis and Design

A thorough analysis and design phase was carried out to ensure the system meets both functional and non-functional requirements.

  • UML Diagrams (Use Case, Activity, Sequence)
  • Normalized Database Schema Design
  • Wireframes and Mockups for the UI/UX
  • Functional Requirements Documentation
  • Non-Functional Requirements (Performance, Usability, Security)

System Implementation

The ManageX system was implemented using modern tools and frameworks to ensure scalability, maintainability, and ease of use for all user personas.

  • Backend developed using Java and Firebase Realtime Database
  • Android application built with Java (Android Studio)
  • Firebase Authentication for secure login and user management
  • Real-time database synchronization between app and backend
  • Role-based access control implemented in the app
  • Continuous testing using JUnit and manual UI testing

Status Reports

The project was closely monitored with regular status updates provided to stakeholders, ensuring transparency and timely progress.

  • Bi-weekly sprint reports shared with the team
  • Milestone reviews to track deliverables
  • Identified risks and mitigation strategies
  • Documented change requests and approvals
  • Client feedback incorporated during development phases
  • Final report showcasing project outcomes and metrics
